Problem Statement

N Prison inmates are standing in a circle waiting to be executed. They are tagged based on their position 1,2,3... so on. Consequently, the Nth inmate stands next to inmate 1. The executioner decides to pardon one of these inmates. His technique for carrying out the execution is simple: he alternately pardons/kills inmates in a cyclic fashion. He pardons the 1st inmate, executes the 2nd, then pardons the 3rd inmate, executes the 4th. If by chance he pardons the Nth inmate he executes the 1st, since they are in a circle. The executioner continues like this until there is only one inmate left. Your job is to predict this lucky prisoner who survives.

Input Format

The first line contains a single integer T denoting the number of test scenarios.

The next T lines each contain an integer N[i] which denotes the number of prisoners rounded up for execution in scenario i.

Constraints

0 < T < 100

0 < N[i] < 10^6

Output Format

Print the lucky prisoner's TAG number. If T scenarios were tested, the output must contain T lines, each line displaying the TAG of the lucky prisoner in that scenario.

Sample Input

2
4
1000

Sample Output

1
977

Explanation

Two scenarios were tested. In the first scenario 4 inmates were rounded up. The executioner pardons 1, kills 2, pardons 3, kills 4 the cycles back, since he had killed 4, he pardons 1, kills 3. Since only one prisoner remains, 1 is the lucky prisoner.

In the second scenario 1000 inmates were rounded up. 977 is the lucky prisoner.
